# Position Layout Template

Job Title — Company Name
📍 Location (or “Remote” / “Hybrid – based in X”)  🗓 Start – End Date
(Optionally add a one-line tagline if your title alone doesn’t show scope)

Example: “Leading data engineering initiatives supporting ML and analytics across global teams.”

## 1. One-line summary (optional but powerful)

A short, assertive description that frames what you own or lead rather than what you were “responsible for.”
Owns design and deployment of scalable data infrastructure enabling company-wide AI workflows.

## 2. Bullet points — impact and substance

Use 3–5 bullets max. Start each with a strong verb and finish with a result or context.
Each should follow this pattern:

!!! [Action Verb] + [Scope or System] + [Concrete Result or Metric] !!!

Examples:

Designed and implemented data pipelines handling 10M+ records daily across multi-cloud environments.

Integrated ML model serving into production APIs, cutting inference latency by 30%.

Led migration from Jenkins to GitHub Actions, reducing deployment time from 45 min to 10 min.

Partnered with analytics and research teams to define data contracts ensuring end-to-end traceability.

💡 Tips:

Don’t use weak verbs like “helped,” “worked on,” or “participated.” Replace with “built,” “led,” “optimized,” “developed,” “deployed,” etc.

Quantify outcomes wherever possible — speed, cost, uptime, scale, or adoption metrics.

Use present tense for your current job, past tense for previous ones.

## 3. Technical / domain keywords block (optional but useful)

Add a concise, keyword-dense line that looks clean:

Tech: Python, Airflow, Spark, AWS, Terraform, Docker, MLflow, Kafka

This increases search visibility without making the bullets messy.

## 4. Achievements or Highlights (optional)

Use this if the role includes notable recognitions or milestones.

Highlights: Presented at internal AI summit · Mentored 3 junior engineers · Delivered 99.9% uptime for critical data platform

Keep this section visual and concise — bullet points or inline dots (·) work best.

## 5. Optional positioning note (for flexible or vague roles)

If your official title was fuzzy or internal, you can clarify it subtly:

Official title: “Innovation Engineer” — functionally acted as DevOps Lead within the data infrastructure team.

This keeps transparency while giving context for your real seniority.

## 6. Spacing & tone

No paragraphs, no “responsible for.”

Use white space — LinkedIn truncates long sections; concise bullets keep you above the fold.

Write in a factual, assertive tone. Avoid filler (“proactive,” “motivated,” “passionate”). Show it through results instead.
